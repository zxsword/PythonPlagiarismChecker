# Python 代码相似度检查工具

这是一个为编程课教师（或任何需要检查代码相似度的人）设计的简单图形界面工具。它可以帮助您快速发现一个文件夹内或您手动选择的多个 Python (`.py`) 文件之间是否存在潜在的抄袭行为。

## ✨ 主要功能

-   **图形用户界面 (GUI)**: 提供一个简单直观的图形界面，无需命令行操作。
-   **两种检查模式**:
    1.  **文件夹模式**: 快速检查一个文件夹内所有 `.py` 文件的相似度。
    2.  **文件模式**: 灵活选择分布在不同位置的多个 `.py` 文件进行检查。
-   **可调相似度阈值**: 您可以自由滑动设置一个百分比，只有相似度高于这个值的文件对才会被认为是可疑的。
-   **清晰的结果列表**: 以表格形式清晰地列出所有相似度超标的文件对和它们的具体相似度。
-   **并排高亮对比**: 在结果列表中选中任何一对文件，点击“对比高亮显示”按钮，即可在一个新窗口中并排查看两份代码，其中相似的代码块会被自动高亮显示，让您对抄袭情况一目了然。
-   **智能代码标准化**: 在比较前，程序会尝试去除代码中的注释、多余空格和不规范的格式，使得比较更加关注代码的核心逻辑，而不是表面差异。

## ⚙️ 运行环境

-   **Python 3**: 本程序基于 Python 3 开发，需要您的系统上已安装 Python 3 环境。
-   **标准库**: 程序完全依赖 Python 内置的标准库 (如 `tkinter`, `difflib`, `ast` 等)，**无需安装任何第三方库**。开箱即用，非常方便。

## 🚀 如何使用

1.  **下载并解压**:
    将整个项目下载或克隆到您的电脑上。

2.  **启动程序** (三选一):
    -   **(推荐) 对于 Windows 用户**: 直接双击项目目录中的 `run.bat` 文件即可启动程序。
    -   **通过命令行**: 打开命令行工具 (如 Terminal, Command Prompt, PowerShell)，进入项目根目录，然后执行以下命令：
        ```bash
        python main.py
        ```
    -   **通过IDE**: 在您喜欢的代码编辑器（如 VS Code, PyCharm 等）中打开项目文件夹，然后直接运行 `main.py` 文件。

3.  **选择检查模式**:
    -   **文件夹模式**: 在程序顶部选择“文件夹模式”，然后点击“选择学生代码文件夹”按钮，选中您存放学生作业的文件夹。
    -   **文件模式**: 选择“文件模式”，然后点击“选择代码文件”按钮，您可以一次性或多次选择需要对比的 `.py` 文件。已选中的文件会显示在下方的列表框中。

4.  **开始检查**:
    -   拖动“相似度阈值”滑块，设定一个您认为合适的百分比。
    -   点击“开始检查”按钮。

5.  **查看结果**:
    -   程序会进行分析，并在下方的“检查结果”表格中列出所有相似度超过阈值的文件对。
    -   底部的状态栏会提示检查的状态（如“检查中...”、“检查完成”等）。

6.  **对比代码**:
    -   在结果表格中，用鼠标单击选中您想详细查看的任意一对文件。
    -   点击表格下方的“**对比高亮显示选中文件**”按钮。
    -   一个新的窗口会弹出，并排显示两份代码，相似部分会被高亮。您可以滚动查看所有代码。

## 🧪 如何运行测试

本项目包含一套单元测试，用于确保核心分析功能的正确性。如果您修改了 `analysis.py` 中的代码，可以通过运行测试来验证您的修改是否引入了错误。

在项目根目录下，执行以下命令：

```bash
python tests/test_analysis.py
```

如果所有测试都通过，您会看到 `OK` 的提示。

```
├── .gitignore              # 告诉 Git 哪些文件不需要版本控制。
├── plagiarism_checker/
│   ├── __init__.py         # Python包的标识文件。
│   ├── analysis.py         # 核心分析模块，包含代码标准化和相似度计算的函数。
│   └── ui/
│       ├── __init__.py     # UI子包的标识文件。
│       ├── app.py          # 主应用窗口的定义和逻辑。
│       └── comparison_window.py # 代码对比窗口的定义和逻辑。
└── tests/
    └── test_analysis.py    # 针对 analysis.py 的单元测试。
```
